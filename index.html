I learned how to code in, and use, Processing while taking Graphics & Virtual Environments at Connecticut College. Here is a fun project for which I had to incooporate animations. You can see that the animation is run on a loop. There are multiple animated parts, incooporating a range of speeds and movements. This was accomplished using a simple animation timer(default speed of frames) and timer step(offsets animation from animation timer); the speed of specific animations and angle of movement were calculated using these two values. Processing is open source, well documented, and easy to code in; which is why we learned this language before tackling Unity later in the course.
